# === Vulnerability Scanners & Pentesting Tools ===
# GOAL: Hard-block these. They are used to find unpatched plugins or weak passwords.
Acunetix           # Professional-grade scanner; used by hackers to map your site's weak points.
Arachni        	   # High-performance scanner for web apps; no legitimate reason for this to hit your site.
Burp               # Standard "Man-in-the-Middle" tool used to intercept and manipulate your site's traffic.
CensysInspect      # Internet-wide scanner; maps your server's IP to your domain for future targeted attacks.
Dirb               # "Brute force" tool that guesses hidden folder names like /admin or /backup.
DirBuster          # Similar to Dirb; a heavy-duty tool for finding files you didn't mean to make public.
dirsearch          # Modern command-line tool that hunts for forgotten .env, .git, or .php config files.
Expanse            # Corporate asset mapper; usually just "noise" in your logs that drains bandwidth.
Feroxbuster        # Extremely fast directory brute-forcer; can cause high CPU spikes on your server.
Fuzz/              # Indicates "fuzzing"—sending random data to your forms to try and crash the site.
Go-http-client     # Default identity for programs in the Go language; almost always a custom attack script.
Gobuster           # High-speed scanner used specifically to find hidden subdomains or directories.
Havij              # Infamous "one-click" tool used to automate SQL Injection (stealing your database).
Indy Library       # Older Delphi-based library; very common in legacy malware and spam tools.
internet-measurement # Usually university or research bots; they provide zero SEO value.
JSScanner/         # Specifically hunts for vulnerabilities in your site's JavaScript files.
masscan            # "The internet's fastest scanner"; used by botnets to find vulnerable IP addresses.
Nessus             # Enterprise-grade vulnerability scanner; if this hits you, a targeted probe is likely.
Netsparker         # Automated web security scanner; used to find SQL injection and XSS holes.
Nikto              # Classic scanner that looks for 6,000+ dangerous files and outdated server software.
Nmap               # The king of network scanners; maps every open door (port) on your server.
Nuclei             # Modern, template-based scanner; used by hackers to find "day-one" exploits.
OpenVAS            # Open-source vulnerability manager; useful for pros, but slows down your site.
Photon/1.0         # A crawler designed to extract URLs, emails, and social media handles for spam.
PostmanRuntime     # Tool used by developers to test APIs; if you aren't a dev, it shouldn't be here.
PressVuln          # Specifically targets WordPress sites to find unpatched plugin or theme bugs.
Qualys             # High-end enterprise security scanner; usually unauthorized when hitting a small site.
ScanNG             # Emerging automated scanner; high-frequency and very noisy in server logs.
SF/                # Generic prefix for many different automated "Security Framework" tools.
sqlmap             # The most dangerous tool for databases; it automates the theft of your user tables.
Tsunami            # Google-developed scanner; powerful, but should only be run by the site owner.
Vega               # Cross-platform security scanner; no benefit to regular human traffic.
Wfuzz              # Used to brute-force web forms, such as your WordPress login page.
WhatWeb            # Fingerprints your site to find exactly which WordPress and PHP versions you use.
WPScan             # The "WordPress Hacking Standard"; it finds holes in your specific plugins.
WPSec              # Similar to WPScan; an automated service that probes for WordPress vulnerabilities.
ZAP/               # OWASP Zed Attack Proxy; a manual tool used to find security flaws.
zgrab              # Used for massive internet-wide scans to identify specific server software.
zgrab2             # The updated version of zgrab; often used to find servers for botnet recruitment.

Python-urllib      # A lower-level Python library; almost exclusively used in bot scripts.

# === Suspicious, Malformed, or Fake User-Agents ===
# GOAL: Hard-block. These are generally clear signs of a bot or hacker trying to hide.
Chrome/100.0.0.0   # Lazy bot-makers use this "perfect" number as a fake, hardcoded identity.
Cloudflare-Requests # Bots trying to look like they are coming from within Cloudflare's network.
Dalvik/            # The engine inside older Android; often used for mobile ad-fraud botnets.
Empty              # If the bot doesn't even provide an ID, it's almost always malicious.
GRequests/         # "Greens" Python library; used for writing custom, high-speed scraping tools.
Headless           # Some bots literally admit they are "headless" (automated) browsers.
#http://           # Identifies bots that accidentally put a web link in their identity string. Could also block legitimate bots.
#https://          # Identifies bots that accidentally put a secure link in their identity string. Could also block legitimate bots.
forms.gle          # Spammers who put a link to a Google Form in their identity to get clicks.
Internet Explorer  # Fake. Real IE uses "MSIE" or "Trident." This is a clear bot sign.
mailto:            # Spammers trying to "leave" an email address in your server logs.
morfeus            # A classic vulnerability scanner identifier from the early 2000s.
Mozlila            # A misspelling of "Mozilla"; a classic "script-kiddie" or low-level bot mistake.
Scraper            # Some bots are honest enough to literally identify as "Scraper."
ShellBot           # A dangerous bot used to "take over" your server via a malicious shell script.
Unknown            # If the identity is literally "Unknown," it is a bot attempting to hide.
User-Agent:        # A bot that accidentally included the header label inside the identity value.
#X11; Linux x86_64  # NOTE: Only block if you want to block legitimate linux users as well.  The Author of this list is one of them...
